Лабораторна робота №1. Бібліотеки Mocha, Chai

Мета: ознайомитись з методикою роботи з бібліотеками Mocha та Chai

Протестовані основні функції бібліотеки Mtrx
1. Створення матриць (основні способи)

new Mtrx() — створення 1×1 матриці з випадковим числом (0..1)
new Mtrx(n) — створення квадратної матриці n×n з випадковими числами
new Mtrx(rows, cols, value) — створення матриці, заповненої сталим значенням
new Mtrx([[...]]) — створення матриці з двовимірного масиву
new Mtrx([числа]) — створення діагональної матриці з масиву чисел
new Mtrx(rows, cols, fn) — створення матриці за функцією (i, j) ⇒ значення
Статичні методи:
Mtrx.zeros(rows, cols) — матриця з нулів
Mtrx.ones(rows, cols) — матриця з одиниць
Mtrx.eye(n) — одинична (ідентична) матриця


2. Перевірки властивостей матриць

Mtrx.isMtrx(obj) — перевірка, чи є об’єкт екземпляром класу Mtrx
Mtrx.equal(m1, m2) — поелементне порівняння двох матриць (повертає матрицю булевих значень)
Mtrx.isDiag(m) — перевірка, чи є матриця діагональною

3. Арифметичні операції (статичні)

Mtrx.add(m1, m2) — додавання двох матриць
Mtrx.mul(m1, m2) — матричне множення
Mtrx.div(m1, m2) — матричне ділення (m1 × inv(m2))

4. Методи екземпляра (основні)

.T() — транспонування матриці
.inv() — обчислення оберненої матриці (для невироджених)
.inv() — перевірка викидання помилки для виродженої (singular) матриці
.mapMtrx(fn) — застосування функції до кожного елемента матриці (аналог map для масивів)

5. Властивості матриці

.rows — кількість рядків
.cols — кількість стовпців
.det — визначник (детермінант) матриці
.rank — ранг матриці

Висновок
У рамках лабораторної роботи було виконано модульне тестування основних функцій бібліотеки Mtrx за допомогою фреймворку Mocha та бібліотеки асерцій Chai. Усього написано та успішно виконано 20 тестових випадків, які охоплюють створення матриць різними способами, перевірку властивостей, арифметичні операції, методи екземпляра та доступ до основних властивостей.
Усі тести пройшли успішно, що підтверджує коректність реалізації основного функціоналу бібліотеки.

Особливості модульного та приймального тестування

Модульне тестування (Unit testing)
Особливості:
Тестує найменші одиниці коду (функції, методи, класи) в ізоляції від решти системи.
Використовує mocks/stubs для заміни залежностей (наприклад, зовнішніх API або баз даних).
Виконується швидко (тисячі тестів за секунди).
Автоматизоване, запускається часто (при кожному коміті).
Виявляє помилки на ранніх стадіях розробки.
Пишуться та підтримуються переважно розробниками.

Приймальне тестування (Acceptance testing)
Особливості:
Виконується в кінці циклу розробки, перед релізом.
Перевіряє, чи продукт відповідає бізнес-вимогам і готовий до використання кінцевим користувачем.
Може бути ручним або автоматизованим (наприклад, Cucumber, Selenium для UI).
Залучає замовника, тестувальників або реальних користувачів.
Тестує систему в цілому, в реальному або наближеному до реального середовищі.
Виявляє проблеми, які не видно на рівні окремих модулів (наприклад, помилки в робочому процесі).

Призначення бібліотек Mocha та Chai

Mocha
Призначення:
Фреймворк для організації, опису та запуску тестів у JavaScript (працює як у Node.js, так і в браузері).
Надає структуру тестів:
describe() — групування тестів у набори (test suites),
it() — опис окремого тестового випадку,
хуки before(), after(), beforeEach(), afterEach() — підготовка та очищення даних,
підтримка асинхронного коду (callbacks, promises, async/await).
Mocha автоматично знаходить і запускає тести командою npm test.
Chai
Призначення:
Бібліотека асерцій (перевірок результатів). Надає зручний і читабельний синтаксис для перевірки очікувань.
Підтримує стилі BDD (expect, should) та TDD (assert).

Разом Mocha + Chai — стандартний набір для модульного тестування в екосистемі Node.js/JavaScript, який ми успішно застосували для перевірки основних функцій бібліотеки Mtrx.
